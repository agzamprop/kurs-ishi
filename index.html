I BOB. AXBOROT TIZIMLARI VA MUZQAYMOQ SANOATI TAHLILI

1.1. Axborot tizimlarining nazariy asoslari

Axborot tizimlari (AT) — bu ma'lumotlarni yig'ish, saqlash, qayta ishlash va uzatish uchun mo'ljallangan tashkiliy-texnik tizimlar majmuasidir. Zamonaviy axborot tizimlari tashkilotlarning strategik maqsadlariga erishishda muhim vosita bo'lib xizmat qiladi1. Ular boshqaruv qarorlarini qabul qilish jarayonini tezlashtiradi va samaradorligini oshiradi.

Axborot tizimlarining asosiy komponentlari quyidagilardan iborat: apparat ta'minoti (serverlar, kompyuterlar, tarmoq qurilmalari), dasturiy ta'minot (operatsion tizimlar, amaliy dasturlar, ma'lumotlar bazasini boshqarish tizimlari), ma'lumotlar (strukturalangan va strukturalanmagan), odamlar (foydalanuvchilar, administratorlar, dasturchilar) va protseduralar (ish qoidalari, ko'rsatmalar).

Axborot tizimlarini tasniflashning bir necha yondashuvlari mavjud. Qo'llanish sohasiga ko'ra ular quyidagi turlarga bo'linadi:

1) Tranzaksion tizimlar (TPS) — kundalik operatsiyalarni qayta ishlash uchun;

2) Boshqaruv axborot tizimlari (MIS) — o'rta bo'g'in rahbarlariga hisobotlar tayyorlash uchun;

3) Qaror qabul qilishni qo'llab-quvvatlash tizimlari (DSS) — murakkab masalalarni hal qilishda yordam berish uchun;

4) Ekspert tizimlar — sun'iy intellekt asosida maslahatlar berish uchun;

5) Ijro axborot tizimlari (EIS) — yuqori bo'g'in rahbarlari uchun strategik ma'lumotlar taqdim etish.

Muzqaymoq do'koni boshqaruv tizimi o'z xususiyatlariga ko'ra tranzaksion tizimlar va boshqaruv axborot tizimlari elementlarini o'zida mujassamlashtiradi. U kundalik savdo operatsiyalarini qayta ishlash bilan birga, boshqaruv qarorlari uchun hisobotlar tayyorlash imkoniyatini ham beradi.

Axborot tizimlarini ishlab chiqishda bir nechta metodologiyalar qo'llaniladi. Shular orasida eng keng tarqalganlari: tizimli tahlil va loyihalash metodologiyasi (SSADM), tezkor dasturiy ta'minot ishlab chiqish (RAD), Agile metodologiyasi va tizimning hayot sikli modeli (SDLC) mavjud.

Tizimning hayot sikli modeli (SDLC) axborot tizimini ishlab chiqishning eng klassik yondashuvlaridan biri hisoblanadi. Bu model quyidagi bosqichlarni o'z ichiga oladi: rejalashtirish, tahlil, loyihalash, ishlab chiqish, testlash, joriy etish va texnik xizmat ko'rsatish. Har bir bosqich o'ziga xos natijalar beradi va keyingi bosqichga o'tish uchun asos yaratadi.

Muzqaymoq do'koni boshqaruv tizimini ishlab chiqishda biz iterativ yondashuvdan foydalanamiz. Bu yondashuv tizimni bosqichma-bosqich yaratish, har bir iteratsiyada yangi funksional imkoniyatlarni qo'shish va mavjud modullalni takomillashtirish imkonini beradi.

Ma'lumotlar bazasi — axborot tizimining eng muhim tarkibiy qismi hisoblanadi. Ma'lumotlar bazasini boshqarish tizimlari (MBBT) ma'lumotlarni saqlash, qidirish, yangilash va o'chirish operatsiyalarini bajaradi. Relyatsion ma'lumotlar bazalari hozirgi kunda eng keng tarqalgan tur hisoblanadi2.

Relyatsion ma'lumotlar bazalari jadvallar (munosabatlar) ko'rinishida tashkil etiladi. Har bir jadval ustunlar (atributlar) va qatorlar (kortejlar) dan iborat. Jadvallar orasidagi bog'lanishlar tashqi kalitlar orqali amalga oshiriladi. SQL (Structured Query Language) — relyatsion ma'lumotlar bazalari bilan ishlash uchun standart til hisoblanadi.

Bizning loyihamizda SQLite relyatsion ma'lumotlar bazasi ishlatiladi. SQLite — bu server talab qilmaydigan, yengil va tezkor MBBT bo'lib, kichik va o'rta hajmdagi loyihalar uchun ideal yechim hisoblanadi. U bitta fayl ko'rinishida saqlanadi va qo'shimcha sozlashlarni talab qilmaydi.

Web-dasturlash texnologiyalari axborot tizimlarini yaratishda keng qo'llanilmoqda. Zamonaviy web-ilovalar uch qavatli arxitektura asosida quriladi: taqdimot qavati (frontend), biznes-logika qavati (backend) va ma'lumotlar qavati (database). Bu arxitektura tizim komponentlarini mustaqil rivojlantirish imkonini beradi.

Backend dasturlash uchun Python dasturlash tili keng imkoniyatlar beradi. Python — bu yuqori darajali, interpretatsiya qilinadigan, ob'ektga yo'naltirilgan dasturlash tilidir. Uning sintaksisi sodda va o'qilishi oson, bu esa uni o'rganish va qo'llashni osonlashtiradi. Python web-dasturlash, sun'iy intellekt, ma'lumotlar fani va boshqa ko'plab sohalarda qo'llaniladi.

Flask — Python uchun mikro web-freymvork bo'lib, u kichik va o'rta loyihalar uchun ideal hisoblanadi. Flask minimalistik yondashuvni qo'llaydi: u faqat asosiy funksionallikni taqdim etadi va qo'shimcha imkoniyatlarni kengaytmalar orqali qo'shish mumkin. Flask ning asosiy xususiyatlari:

— Jinja2 shablon mexanizmi — dinamik HTML sahifalar yaratish uchun;

— Werkzeug WSGI kutubxonasi — HTTP so'rovlarni qayta ishlash uchun;

— URL marshrutlash — URL manzillarni funksiyalarga bog'lash uchun;

— Kengaytmalar tizimi — Flask-SQLAlchemy, Flask-Login va boshqalar.

Frontend texnologiyalari orasida HTML5, CSS3 va JavaScript asosiy vositalar hisoblanadi. HTML5 veb-sahifalarning strukturasini belgilaydi, CSS3 ularning ko'rinishini shakllantiradi, JavaScript esa interaktivlikni ta'minlaydi. Bootstrap — CSS freymvorki bo'lib, responsiv va zamonaviy foydalanuvchi interfeyslarini tez yaratish imkonini beradi.

Axborot xavfsizligi ham tizimni ishlab chiqishda muhim jihat hisoblanadi. Web-ilovalar uchun asosiy xavfsizlik tahdidlari: SQL in'ektsiya, XSS (Cross-Site Scripting), CSRF (Cross-Site Request Forgery) va autentifikatsiya zaifliklarini o'z ichiga oladi. Tizimni ishlab chiqishda ushbu tahdidlardan himoyalash mexanizmlari joriy etilishi lozim.

API (Application Programming Interface) — dasturlar orasidagi aloqani ta'minlovchi interfeys hisoblanadi. REST (Representational State Transfer) arxitekturasi zamonaviy web-xizmatlarni yaratishda standart yondashuv bo'lib, HTTP protokoli ustida ishlaydi. RESTful API quyidagi printsiplarga asoslanadi:

— Mijoz-server arxitekturasi — frontend va backend ajratilgan;

— Holatni saqlamaslik (Stateless) — har bir so'rov mustaqil;

— Keshlanish — javoblar keshlanishi mumkin;

— Yagona interfeys — standart HTTP metodlari (GET, POST, PUT, DELETE).

Tizimni loyihalashda UML (Unified Modeling Language) diagrammalari muhim rol o'ynaydi. UML — bu dasturiy ta'minot tizimlarini vizualizatsiya qilish, loyihalash va hujjatlashtirish uchun standart modellashtirish tili. Asosiy UML diagrammalari: foydalanish holatlari diagrammasi (Use Case), sinf diagrammasi (Class Diagram), ketma-ketlik diagrammasi (Sequence Diagram) va holatlar diagrammasi (State Diagram).

Dasturiy ta'minotni testlash — tizim sifatini ta'minlashda zarur jarayon. Testlashning asosiy turlari: birlik testlash (unit testing) — alohida modullarni tekshirish, integratsion testlash — modullar orasidagi o'zaro ta'sirni tekshirish, va tizimli testlash — butun tizimni kompleks tekshirish. Python da pytest va unittest kutubxonalari testlash uchun keng qo'llaniladi.

Versiyalarni boshqarish tizimlari (VCS) dasturiy ta'minotni ishlab chiqishda kod o'zgarishlarini kuzatib borish imkonini beradi. Git — eng mashhur taqsimlangan versiyalarni boshqarish tizimi bo'lib, u jamoaviy ishlashni osonlashtiradi va kod tarixini saqlaydi. GitHub — Git repozitoriylarini masofaviy saqlash va hamkorlik platformasi.

Xulosa qilib aytganda, axborot tizimlari nazariyasi va zamonaviy web-texnologiyalar muzqaymoq do'koni boshqaruv tizimini ishlab chiqish uchun mustahkam asos yaratadi. Keyingi paragraflarda ushbu bilimlarning amaliy qo'llanilishi tahlil qilinadi.

Avtomatlashtirishning iqtisodiy samaradorligini baholash uchun ROI (Return on Investment) ko'rsatkichidan foydalaniladi. Muzqaymoq do'koni uchun boshqaruv tizimini joriy etishning iqtisodiy samarasi quyidagi omillar hisobiga yuzaga keladi:

@reports.route('/reports/sales')
@login_required
def sales_report():
    period = request.args.get('period', 'monthly')
    if period == 'daily':
        start = datetime.utcnow() - timedelta(days=1)
    elif period == 'weekly':
        start = datetime.utcnow() - timedelta(weeks=1)
    else:
        start = datetime.utcnow() - timedelta(days=30)
    
    sales = db.session.query(
        func.date(Order.created_at).label('date'),
        func.sum(Order.total_amount).label('total'),
        func.count(Order.id).label('count')
    ).filter(
        Order.created_at >= start,
        Order.status == 'bajarildi'
